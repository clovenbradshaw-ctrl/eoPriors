# Basis candidates

Proposed-but-not-activated exemplar bases land here as one JSON file per
proposal, named `<basis_id-without-the-exemplar-basis-prefix>.json`, matching
`schemas/exemplar-basis.schema.json` with `"status": "proposed"`.

A file here is a **record of what `exemplar.basis.selected` proposed**, kept
for human review — never itself a live basis. Promoting one to
`config/exemplar-basis/active.json` happens only through the governance path
in SPEC.md §5.5: a human-reviewed PR, never a batch, never code running here.

This directory is empty until the first basis proposal is reviewed.
